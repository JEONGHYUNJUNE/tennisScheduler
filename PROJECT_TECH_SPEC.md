# ONS Tennis 프로젝트 기술 명세

이 문서는 ONS Tennis 웹앱 내부 기능 구조, 데이터 흐름, Supabase 구성, 주요 구현 판단을 정리하기 위한 개발자용 문서입니다. `README.md`와 분리해서 관리하며, 앞으로 기능이 늘어날 때마다 섹션을 추가해 프로젝트 내부 스펙 문서처럼 사용할 수 있습니다.

## 채팅 시스템

### 기능 개요

채팅 기능은 멤버 간 1:1 실시간 대화를 지원합니다. 한 멤버가 다른 멤버에게 채팅을 요청하고, 상대가 채팅방에 들어오면 방이 활성화됩니다. 메시지, 사진, 동영상, 커스텀 이모티콘, 답장, 공지, 읽음 표시, 반응 이모티콘, 채팅 검색, 네이버 검색 공유, 푸시 알림이 React 상태 관리와 Supabase 테이블, RPC 함수, Storage, Realtime, Edge Function을 조합해 동작합니다.

### 주요 프론트엔드 파일

- `src/pages/ChatRoomPage.jsx`
  - 채팅방 화면 전체를 담당합니다.
  - 메시지 렌더링, 입력창, 사진/영상 전송, 이모티콘 패널, 답장, 공지, 반응 이모티콘, 채팅 검색, 네이버 검색 공유, 이미지 미리보기 연결, 읽음 표시 UI를 포함합니다.
- `src/services/chatService.js`
  - Supabase와 통신하는 채팅 서비스 계층입니다.
  - 채팅방 조회, 메시지 조회/전송, 사진/영상 업로드, 커스텀 이모티콘, 반응 이모티콘, 읽음 처리, 채팅방 상태 변경, Realtime 구독을 담당합니다.
- `src/services/notificationService.js`
  - 앱 내 알림 목록을 조회하고 필터링합니다.
  - 채팅 메시지/채팅 반응 알림은 푸시용으로만 쓰고 앱 안 종모양 알림 리스트에서는 숨깁니다.
- `src/components/ImageLightbox.jsx`
  - 채팅 및 게시글 이미지 클릭 시 주소창으로 이동하지 않고 앱 안에서 크게 보여주는 전체 화면 이미지 뷰어입니다.
  - 기기에 저장 기능도 이 컴포넌트에서 처리합니다.
- `src/styles.css`
  - 채팅방 레이아웃, 말풍선, 입력창, 키보드 대응, 반응 이모티콘 칩, 이모티콘 패널, 이미지/영상 메시지, 모바일 대응 스타일을 포함합니다.

### Supabase 데이터 구조

채팅 기능은 주로 아래 테이블을 사용합니다.

- `chat_rooms`
  - 1:1 채팅방의 기본 정보를 저장합니다.
  - 요청자, 수신자, 방 상태, 읽음 시각, 접속 시각, 현재 공지 메시지, 종료 상태 등을 관리합니다.
- `chat_messages`
  - 채팅 메시지를 저장합니다.
  - 텍스트, 이미지, 동영상, 이모티콘, 시스템 메시지, 답장 대상, 공지 대상, 파일 메타데이터를 포함합니다.
- `chat_message_reactions`
  - 메시지별 반응 이모티콘을 저장합니다.
  - 한 멤버는 한 메시지에 하나의 현재 반응만 유지합니다.
  - 기본 이모지 반응과 커스텀 이모티콘 반응을 모두 지원합니다.
- `chat_custom_stickers`
  - 사용자가 만든 커스텀 이모티콘 메타데이터를 저장합니다.
  - 실제 이미지 파일은 Supabase Storage에 저장합니다.
- `ot_notifications`
  - 푸시 알림 발송을 위한 알림 row를 저장합니다.
  - 채팅 메시지 및 채팅 반응 알림은 이 테이블에는 저장되지만, 앱 내 종모양 알림 리스트에서는 숨깁니다.

### Realtime 구성

채팅방은 Supabase Realtime을 사용합니다. 메시지, 방 상태, 반응 이모티콘은 각각 별도 채널로 구독합니다.

- `chat_messages`
  - `INSERT` 이벤트를 구독합니다.
  - 새 메시지가 들어오면 이벤트 payload만 바로 쓰지 않고, `getChatMessage()`로 다시 hydrate해서 작성자, 답장, 미디어, 반응 정보가 포함된 완성 메시지로 렌더링합니다.
- `chat_rooms`
  - `UPDATE` 이벤트를 구독합니다.
  - 채팅방 상태, 읽음/접속 시각, 공지 변경을 반영합니다.
- `chat_message_reactions`
  - 반응 이모티콘 변경 이벤트를 구독합니다.
  - 반응이 생기거나 바뀌면 해당 메시지만 다시 조회해서 기존 메시지 목록에 병합합니다.

반응 이모티콘을 실시간으로 보이게 하려면 `chat_message_reactions` 테이블도 Supabase Realtime publication에 포함되어야 합니다.

```sql
alter table public.chat_message_reactions replica identity full;

do $$
begin
  alter publication supabase_realtime add table public.chat_message_reactions;
exception
  when duplicate_object then null;
end $$;
```

메시지, 방 상태, 반응 구독을 분리한 이유는 반응 테이블 쪽 문제가 생겨도 핵심 메시지 수신 흐름이 같이 흔들리지 않게 하기 위해서입니다.

### 읽음 및 접속 상태 처리

채팅 읽음 처리는 메시지마다 별도 읽음 row를 만들지 않고, 채팅방 단위 timestamp로 처리합니다.

- `chat_rooms`에 요청자/수신자의 마지막 읽음 시각과 마지막 접속 시각을 저장합니다.
- 사용자가 채팅방을 보고 있으면 주기적으로 읽음/접속 상태를 갱신합니다.
- 내가 보낸 메시지는 상대가 아직 읽지 않았으면 카카오톡처럼 `1`을 표시합니다.
- 상대가 읽으면 `1`이 사라집니다.
- 채팅 목록의 안 읽은 수는 내 마지막 읽음 시각 이후에 생성된 상대 메시지를 기준으로 계산합니다.

이 방식은 1:1 채팅에서는 구조가 가볍고, 메시지마다 읽음 테이블을 별도로 관리하지 않아도 됩니다.

### 메시지 전송 흐름

메시지 전송은 화면에서 직접 테이블에 insert하지 않고 서비스 함수/RPC를 통해 처리합니다.

기본 흐름은 다음과 같습니다.

1. 사용자가 텍스트, 사진, 영상, 이모티콘 등을 입력하거나 선택합니다.
2. `chatService.js`의 전송 함수가 Supabase에 메시지를 저장합니다.
3. 전송 성공 시 반환된 메시지를 화면에 즉시 추가합니다.
4. Realtime INSERT 이벤트로 같은 메시지가 다시 들어올 수 있으므로 `mergeMessages()`가 message id 기준으로 중복 없이 병합합니다.

`mergeMessages()`는 같은 메시지에 대해 한쪽 데이터가 더 자세한 경우, 답장 정보 등을 보존하면서 병합합니다.

### 사진 및 동영상 전송

사진과 동영상은 Supabase Storage에 저장하고, 메시지 row에는 파일 경로와 메타데이터를 저장합니다.

- 사진
  - 사용자가 이미지를 선택합니다.
  - 전송 전 미리보기 팝업이 열립니다.
  - 편집을 누르면 간단한 편집 UI가 열립니다.
  - 사진 편집은 외부 라이브러리가 아니라 브라우저 기본 `canvas` API를 사용합니다.
  - 사용자는 바로 전송하거나 간단한 드로잉/조정 후 전송할 수 있습니다.
- 동영상
  - 사용자가 영상을 선택합니다.
  - 설정한 용량/길이 제한을 통과한 영상만 전송합니다.
  - 영상 편집은 구현하지 않았습니다. iOS, Android, PWA, 웹 호환성을 우선해서 안정적인 전송만 지원합니다.
- 이미지 미리보기
  - 이미지를 클릭하면 파일 URL로 직접 이동하지 않고 앱 안 `ImageLightbox`로 크게 보여줍니다.
  - 팝업 안에서 기기에 저장할 수 있습니다.

### 이모티콘 및 커스텀 스티커

채팅에는 기본 이모티콘과 사용자가 직접 만든 커스텀 이모티콘이 있습니다.

- 기본 이모티콘
  - 이모티콘 패널 상단에 몇 개의 기본 이모티콘을 고정해서 보여줍니다.
- 커스텀 이모티콘
  - 사용자가 이미지를 선택해 직접 만들 수 있습니다.
  - 편집은 브라우저 `canvas` API를 사용합니다.
  - GIF는 가능하면 GIF 상태를 유지합니다.
  - 커스텀 이모티콘 수가 많아지면 패널 안에서 페이지를 나눠 보여줍니다.

이모티콘 메시지는 같은 이미지를 매번 새로 업로드하지 않도록, 가능하면 기존 Storage 경로를 재사용하는 방식으로 보냅니다. 이렇게 하면 채팅을 많이 해도 Storage 사용량이 불필요하게 늘어나는 것을 줄일 수 있습니다.

### 반응 이모티콘

반응 이모티콘은 `chat_message_reactions` 테이블에 저장합니다.

- 사용자가 말풍선을 길게 누르면 액션 메뉴가 열립니다.
- 기본 이모지 또는 커스텀 이모티콘을 반응으로 남길 수 있습니다.
- 한 사용자는 한 메시지에 하나의 현재 반응만 가집니다.
- 반응 선택 시 UI는 낙관적 업데이트를 사용합니다.
  - 액션 메뉴를 즉시 닫습니다.
  - 말풍선 아래에 반응을 먼저 표시합니다.
  - 반응 칩이 크게 나타났다 자리 잡는 애니메이션을 보여줍니다.
  - 서버 저장은 백그라운드에서 진행합니다.
  - 성공하면 서버에서 다시 조회한 메시지로 확정 반영합니다.
  - 실패하면 해당 메시지를 다시 불러와 서버 상태로 되돌립니다.

커스텀 이모티콘 반응은 원본 이미지가 커도 반응 칩 안에서는 작은 크기로 고정합니다. 그래서 일반 이모티콘 메시지 크기와 반응 이모티콘 크기가 섞이지 않습니다.

반응 알림은 메시지 작성자가 현재 채팅방에 없을 때만 생성합니다. 새 채팅 메시지 알림과 같은 기준을 따르며, 앱 내 종모양 알림 리스트에는 표시하지 않습니다.

### 답장 기능

답장은 원본 메시지 내용을 복사해서 저장하지 않고, 답장 대상 메시지 id를 참조하는 방식입니다.

- 메시지 row에 답장 대상 id를 저장합니다.
- 프론트에서 답장 대상 메시지 정보를 hydrate해서 보여줍니다.
- 답장 말풍선에는 원본 작성자와 미리보기 텍스트를 표시합니다.
- 답장 영역을 누르면 현재 로드된 메시지 안에서 원본 메시지 위치로 이동합니다.

### 공지 기능

채팅 메시지 하나를 방 상단 공지로 지정할 수 있습니다.

- `chat_rooms`에 현재 공지 메시지 id를 저장합니다.
- 공지 변경은 `chat_rooms` Realtime UPDATE 이벤트로 반영합니다.
- 공지를 등록하면 채팅방 안에 안내 메시지를 하나 자동으로 보냅니다.
  - 예: `정현준님이 공지로 등록했습니다.`
- 이 안내 메시지는 답장 UI와 비슷하게 공지 대상 메시지를 참조해서 보여줍니다.

### 채팅방 검색

채팅방 안에서 메시지를 검색할 수 있습니다.

- 검색어는 메시지 본문 기준으로 조회합니다.
- 검색 결과를 누르면 해당 메시지 주변 맥락을 불러옵니다.
- 다시 맨 아래로 이동하면 최신 메시지 흐름으로 돌아옵니다.
- 검색 중에는 예전 메시지 맥락과 최신 realtime append가 섞이지 않도록 별도 상태로 관리합니다.

### 네이버 검색 공유

채팅 입력창에서 `#` 검색 공유 기능을 지원합니다.

- 사용자가 `#검색어` 형태로 검색 공유를 시작할 수 있습니다.
- 검색은 브라우저에서 직접 Naver API를 호출하지 않고 `naver-search` Supabase Edge Function을 통해 처리합니다.
- Naver API 키는 프론트 번들에 들어가지 않고 서버 측 Edge Function에만 둡니다.
- 사용자가 검색 결과를 선택하면 검색 결과 카드 형태로 채팅 메시지에 저장합니다.
- 외부 링크를 보고 돌아왔을 때 검색 패널 상태가 최대한 유지되도록 `sessionStorage`에 임시 draft를 저장합니다.

### 푸시 알림

푸시 알림은 `ot_notifications` row 생성 후 `send-push-notification` Supabase Edge Function을 통해 발송합니다.

채팅 관련 알림 타입은 다음과 같습니다.

- `chat_requested`
- `chat_message_created`
- `chat_message_reaction_created`

채팅 메시지와 채팅 반응 알림은 앱 내 종모양 알림 목록에서는 숨깁니다. 대신 푸시 알림과 딥링크 이동에는 사용합니다.

채팅 관련 푸시 딥링크는 아래 형태로 이동해야 합니다.

```text
/chats/:chat_room_id
```

### Edge Function 구성

채팅과 관련된 주요 Supabase Edge Function은 다음과 같습니다.

- `send-push-notification`
  - `ot_notifications`에 생성된 알림 row를 바탕으로 웹 푸시를 발송합니다.
  - 채팅방 딥링크를 포함합니다.
  - 푸시 제목/내용/딥링크 로직을 바꾸면 재배포가 필요합니다.
- `naver-search`
  - 네이버 검색 API를 서버 측에서 호출합니다.
  - Naver API 키를 브라우저에 노출하지 않기 위해 사용합니다.

### 모바일 및 PWA 고려사항

현재 앱은 네이티브 앱이 아니라 웹/PWA 기반이므로 일부 네이티브 채팅앱과 완전히 동일하게 처리하기 어려운 부분이 있습니다.

- iOS Safari/PWA의 키보드 보조 바는 웹에서 완전히 제거할 수 없습니다.
- 채팅 입력창은 키보드 바로 위에 최대한 붙도록 조정되어 있습니다.
- 채팅방 안에서는 하단 앱 네비게이션을 숨겨 화면 공간을 확보합니다.
- 말풍선 길게 누르기 메뉴는 직접 구현되어 있으며, 이미지 미리보기, 모바일 터치 이벤트, 브라우저 기본 context menu와 충돌하지 않게 조정되어 있습니다.
- 이미지 저장 동작은 iOS, Android, Safari, Chrome, 설치형 PWA 환경에 따라 세부 동작이 다를 수 있습니다.

### 실패 및 복구 전략

채팅 기능은 선택 기능이 실패해도 핵심 메시지 기능이 최대한 유지되도록 구성합니다.

- 반응 이모티콘 저장 실패 시 해당 메시지만 다시 조회해 서버 상태로 복구합니다.
- 최신 스키마가 아직 반영되지 않은 환경에서는 일부 select 구문을 fallback 형태로 다시 시도합니다.
- Realtime payload는 바로 화면에 쓰지 않고, 필요한 경우 메시지를 다시 조회해서 완성된 데이터로 렌더링합니다.
- 낙관적 UI는 실패 시 되돌리기 쉬운 기능에만 적용합니다.

## 추후 문서화할 항목

아래 기능들은 이후 같은 문서에 이어서 정리할 예정입니다.

- 소통 게시판
- 테니스 다이어리 및 그룹 다이어리
- 일정 등록 및 일정 상세
- 푸시 알림 타입/딥링크 매트릭스
- Supabase Storage 버킷 및 정책
- PWA 설치 및 업데이트 흐름
- 디자인 시스템 및 모바일 레이아웃 규칙
